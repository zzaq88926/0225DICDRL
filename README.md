# 個人網頁實作紀錄

## 專案簡介

這是我在**國立中興大學深度強化學習課程**中實作的個人靜態網頁，用來練習 **Git 版本控制**與 **GitHub Pages 部署**的完整流程。  
透過這次實作，我對從寫程式到正式上線的 DevOps 流程有了更具體的掌握。

## 實作目標

- 建立一個具備現代設計風格的個人介紹網頁
- 熟悉 Git 基本操作（`init`, `add`, `commit`, `push`）
- 透過 GitHub Pages 將靜態網站公開上線

## 頁面功能

| 功能 | 說明 |
|------|------|
| 即時時鐘 | 使用 JavaScript `setInterval` 每秒更新，顯示當前時間與日期 |
| 個人資訊 | 顯示姓名、研究方向與自我介紹 |
| 技能標籤 | 全端網頁開發、LLM 相關技術 |
| GitHub 連結 | 外部連結至個人 GitHub |
| 響應式設計 | 支援桌機與手機瀏覽 |

## 技術架構

- **HTML5** – 語義化結構
- **CSS3** – 玻璃態（Glassmorphism）深色主題、漸層動畫背景、`backdrop-filter`
- **Vanilla JavaScript** – 即時時鐘邏輯
- **Google Fonts (Inter)** – 現代無襯線字型
- **GitHub Pages** – 靜態網站免費託管

## 實作心得

透過這次實作，我走過了完整的流程：

1. **撰寫前端程式碼**：理解 HTML 結構與 CSS 視覺設計的搭配
2. **Git 操作**：從 `git init` 建立本地倉庫，到 `git remote add`、`git push` 推送至遠端
3. **GitHub Pages 部署**：在 Settings → Pages 設定分支，讓靜態網頁正式對外公開

整個流程讓我對「寫程式 → 版本管理 → 部署上線」有了實際體驗，也加深了我對全端開發與 LLM 應用整合方向的興趣。
